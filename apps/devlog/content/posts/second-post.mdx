# 규율은 언젠가 무너진다

아키텍처 경계를 "사람이 지키는 규율"에 맡기면 언젠가 무너진다.
바쁠 때, 피곤할 때, 딱 한 번만 하는 예외가 쌓인다. 그래서 경계를 **도구로 강제**하기로 했다.

## 역할을 겹치지 않게 나눈다

한 가지 일은 한 도구가 담당한다.

1. **Steiger** — FSD 레이어 방향과 public API 경계
2. **knip** — 죽은 코드(미사용 export·파일)
3. **ESLint** — 그 사이의 코드 품질
4. **Prettier** — 순수 포맷

이렇게 나누면 도구끼리 싸우지 않는다. 예를 들어 import 경로 스타일은 Steiger가,
정렬은 ESLint가 맡아 서로 침범하지 않는다.

## 죽은 슬라이스를 잡는 방법

knip 설정에서 슬라이스 barrel을 진입점으로 지정하지 않는다.

```ts
// 라우트에서 도달할 수 없는 슬라이스는 미사용으로 잡힌다
export const config = {
  entry: ['src/app/**/{page,layout}.tsx'],
};
```

이러면 "필요할 때만 만든다"가 규율이 아니라 도구로 강제된다.
빈 슬라이스를 미리 만들면 `no-segmentless-slices`가 즉시 막는다.

## 배운 것

경계를 도구로 강제하는 것은 규율에 의존하는 것보다 한 걸음 앞선 선택이다.
그리고 그 도구들이 실제로 위반을 잡아줄 때, 왜 이렇게 세팅했는지 스스로 설명할 수 있게 된다.
